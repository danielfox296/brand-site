# Deployment

## How it works

The site deploys to **GitHub Pages** at **entuned.co** via GitHub Actions.

- **Repo:** https://github.com/danielfox296/brand-site
- **Workflow:** `.github/workflows/deploy.yml`
- **Trigger:** Push to `main` branch (also manual via workflow_dispatch)
- **DNS:** GoDaddy → GitHub Pages (CNAME file in repo root)

## To deploy

```bash
cd ~/Desktop/entuned/website
python3 build.py              # Rebuild HTML from _src/ sources
git add -A
git commit -m "description of changes"
git push origin main
```

GitHub Actions will:
1. Run `python3 build.py` (rebuilds from source in CI too)
2. Build the player app from `entuned-player` repo and drop it into `/play/`
3. Stage everything into `_site/` and deploy to GitHub Pages

## What gets deployed

The workflow copies these into `_site/`:
- All root `.html` files (generated by build.py)
- `styles.css`
- `CNAME`
- `img/` directory
- `audio/` directory
- `blog/` directory
- `play/` directory (built from entuned-player repo)

## Testing locally

```bash
cd ~/Desktop/entuned/website
python3 -m http.server 8000
# Visit http://localhost:8000
```

Use the HTTP server, not `file://` — the audio player needs HTTP to load MP3 files.

## Important notes

- **Never edit root `.html` files directly.** They get overwritten by `build.py`. Edit source in `_src/`.
- **The player app is a separate repo** (`entuned-player`). It gets pulled in during CI. To update the player, push to that repo — a `repository_dispatch` event triggers a brand-site rebuild.
- **See `ARCHITECTURE.md`** for the full source file structure and build process.
